# Prompt Hardening: Phase 1 Addendum

**Date:** 2026-03-01

**Motivation:**
To improve the legal quality, regulatory density, and strictness of the drafts generated by Engine V2. The LLM naturally defaulted to explanatory, pedagogical prose with generic legal principles, omitting precise operative rules and deductive logic (Fact → Rule → Subsumption → Consequence).

**Technical Summary:**
The core system prompt (`PromptGenerator.js`) was hardened by modifying four core instruction blocks under the Strict Envelope Model:
1. `STRUCTURAL_MANDATE`: Forced a closed 4-step argumentative structure (Fact, Specific Rule, Subsumption, Consequence) for every legally operative claim.
2. `FACTUAL_FIDELITY`: Added a requirement to anticipate and neutralize the most probable procedural or substantive objections within the subsumption phase for adversarial documents.
3. `NEGATIVE_CONSTRAINTS`: Banned abstract explanations, pedagogical tone, implicit "consult a lawyer" clauses, and treatises. Commanded direct execution.
4. `INSTRUCTION`: Formulate highly specific, surgically precise, and immediate executable procedural requests linked to prior subsumptions.

**Baseline Metrics:**
* Low regulatory density (broad doctrine rather than explicit citations).
* Heavy reliance on generic academic/advisory language.
* Disconnected syllogisms (facts and rules stated, but subsumption bypassed).
